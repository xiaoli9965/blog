              # 1. Zookeeper 简介

              • 介绍: Apache ZooKeeper 是一种用于分布式应用程序的高性能协调服务。
              • 特点: 数据存在内存中，类似文件系统的树形结构(文件和目录)，高吞吐量和低延迟，集群高可靠。
              • 作用: 基于 zookeeper 可以实现分布式统一配置中心、服务注册中心，分布式锁等功能的实现；

              使用 Java 语言编写，需要 Java 环境

              # 1.1. 数据结构

              类似 Unix 文件系统树形结构，每个目录称为Znode节点; 每个节点既可以存储数据也可以作为目录节点

              # 1.2. 节点类型

              每个节点最多存放1M的数据.同节点下的子节点名称不能相同

              • 持久节点
                • create /app1 666
              • 顺序节点
                • create -s /app1/cp 888
              • 临时节点(创建的客户端断开连接就消失)
                • create -e /app2 888
              • 临时顺序
                • create -e -s /app1/ 888
              • TTL 节点(3.5+)
              • 容器节点(Container容器节点下面无子节点在一定的时间后会被删除)

              # 1.3. 会话机制

              • 一个客户端连接一个会话，由 zk 分配唯一会话 id
              • 客户端以特定的时间间隔发送心跳以保持会话有效； tickTime
              • 超过会话超时时间未收到客户端的心跳，则判定客户端死了；(默认 2 倍 tickTime)
              • 会话中的请求按 FIFO 顺序执行。

              # 1.4. Watch 监听机制

              类型:

              • data watch: 监听 数据变更
              • child watch: 监听子节点变化

              特性

              • 一次性触发： watch 触发后即被删除。要持续监控变化，则需要持续设置 watch
              • 有序性： 客户端先得到 watch 通知，后才会看到变化结果

              # 几大特性

              • 扩展性
                • 架构设计
                  • Leader
                  • Follower
                  • Observer
                • 读写分离
                • 分布式
              • 可靠性:
                • 数据的可靠性
                  • 一致性 paxos
                • 服务的可靠性
                  • 在Leader挂了的时候有一段不可用的时间
                  • 但是可以快速进行选Leade,恢复可用

              # ZAB 原子广播协议

              原子的含义: 要么成功,要么失败 广播的含义: 分布式,多节点的.. 广播不代表全部都能听到(过半通过)

              关键词:

              • 两阶段提交,
              • 队列, FIFO 顺序性
              • 写log, 写日志是写到磁盘,zk的数据是存在内存.
