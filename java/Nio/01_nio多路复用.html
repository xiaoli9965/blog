          # IO的分类

          • 网络IO
          • 磁盘IO
          • 输入设备的IO

          # BIO

          阻塞IO, accept和recv都是阻塞的 一个连接都需要开一个线程来处理

          优势: 可以接收很多的连接

          问题: 线程太多,cpu要频繁的切换会浪费cpu的调度 线程也会造成内存的浪费,

          根源 BLOCKING阻塞: accpet , recv 解决方法使用 NONBOLCKING(NIO) 非阻塞

          # NIO

          NONBOLCKING 优势: 规避了多线程的问题,只需要一个线程就处理完成很多的连接

          弊端: 存在C10K问题 假设1W个连接,只有一个发来数据. 每次循环你都必须要想内核发送1W次的recv调用 其中9999次是无用的循环,浪费了系统资源和消耗的时间(涉及到频繁的用户态内核态切换)

          # 多路复用器 (Select / poll / epoll)

          Select,poll,epoll 都叫多路复用器

          多路复用器解决C10K问题,在循环遍历一万个recv之前,先调用 select来获取那些socket有了数据O(1), 然后对有响应的进行处理 O(m)

          多路复用器只是给了你状态,告诉你那些socket发来了数据, 读取数据还是需要自己去读

          优势:

          • 通过一次系统调用,吧fds传给内核,内核进行遍历;
          • 相比C10K,节省了多次的用户内核态切换

          # select

          select 内部有个限制,上限一次只能读取1024个文件描述符.(程序写死 不可修改)

          # poll

          poll没有读1024个文件描述符的限制, 读多少个根据系统限制limit来的. 可以自己设置

          # select,poll的弊端

          ** 弊端:**

          • 每次都要重新传入fd, 解决方案内核开辟空间保fd
          • 每次 select/poll时都需要重新全量遍历fd(内核操作也会耗时), 这个过程是阻塞的 外部调用也需要等待遍历结束
          • 涉及到知识 (中断,callback,增强等)

          # epoll 🔥

          如果操作系统支持epoll, JDK会优先选择epoll... 可以自己设置收到切回 select

          epoll节省了遍历fd的成本:

          • 牺牲内核的空间,来换取调用的时间
          • fd放在内核中的一个空间中,从socket连接到断开只会放进去一次. app空间中有一份,内核中有一份
          • 内核中有一个返回区,当socket从外界发来数据包时,内核会将fd的副本放到返回区中
            • 程序只需要从返回区中获取fd状态就可以了
          • app程序只关注IO状态, 返回区的内容才是程序想要的

          epoll核心的2类, 内核提供的api:

          • epoll_create: 在内核开辟一块系统空间,存放epdf... 在整个程序中调用一次就行了
          • epoll_ctl: 有add,mod,del参数
            • 方法4个参数的说明 epoll_ctl('epoll_create创建的空间', '具体的操作 ADD,DEL,MOD', fd文件描述符, '事件:READ')
            • 将一个普通的fd转换为为epfd, 比如将 df添加到 ep空间中
            • 成为epfd之后,当网卡有数据发到fd时, cpu会将epfd放到 返回区中
          • epoll_wait: 用于获取返回区的epfd, 可以传入timeout等待时间
            • 最优 O(1)的复杂度

          epoll相对于 (select,poll)多了两个 epoll_create,epoll_ctl api系统调用

          其实也是同步的IO模型,调用epoll_awit之后,需要调用accpet进行同步读IO

          # 无论是BIO,NIO在读IO数据的时候都是同步模型

          区别有阻塞有非阻塞

          面试题: 是否通过多路复用器就可以快速读取IO数据? 否, 多路复用器只是告诉你了那些文件描述符可读. 读数据需要自己调用recv去读取
