      # 关键点速记

      • 不传参数创建,默认初始容量16
      • 创建时指定初始容量,内部会进行计算. 最终的值会比传入的大,最终算出来的树绝对是2幂次方
        • tableSizeFor(initialCapacity + (initialCapacity >>> 1) + 1))
      • 创建时初始容量计算完毕后会赋值给sizeCtl变量
      • 如果数组长度小于64,即使链表长度达到8也会不会变成树,会走数组扩容逻辑

      # 存储数据的结构

      • 和普通Map的结构基本一致
      • 数组+链表+红黑树
      • 存储数据的结构是 Node对象, key,value, 指向下一个节点的node指针next
        • next的作用是hash冲突之后生成链表用的
      static class Node<K,V> implements Map.Entry<K,V> {
        final int hash;
        final K key;
        volatile V val;
        volatile Node<K,V> next;
}

      # 💥 SizeCtl含义 (简称:SC)

      private transient volatile int sizeCtl;
      1. sizeCtl为0的时候,代表数据未初始化,且数组的初始化容量为16
      2. sizeCtl为正数, 如果数据组未初始化,那么记录的是数组的初始容量,如果数组已经初始化,那么其记录的是数组的扩容阈值
        1. 阈值计算方式 ( 初始容量 * 0.75)
      3. 为-1, 表示数组正在初始化
      4. 小于0并且不是-1,表示数组正在扩容, -(1+n)就是正在进行扩容的线程数

      # 代码逻辑

      # new

          public ConcurrentHashMap(int initialCapacity) {
        if (initialCapacity < 0)
            throw new IllegalArgumentException();
        // 给的初始容量 + 初始容量/4+1
        int cap = ((initialCapacity >= (MAXIMUM_CAPACITY >>> 1)) ?
                   MAXIMUM_CAPACITY :
                   tableSizeFor(initialCapacity + (initialCapacity >>> 1) + 1));
        this.sizeCtl = cap;
    }

      创建时指定初始容量,内部会进行计算. 最终的值会比传入的大,最终算出来的树绝对是2幂次方

      new 的时候只是计算了一下 siziclt, 并没有实际初始化数组

        return (h ^ (h >>> 16)) & HASH_BITS;

      1. key 和 val 都不允许为空
      2. 计算hashcode的值,算出来一定是一个正数.
      3. HASH_BITS 扰动参数使算出来一定是正数

      # initTable方法

      第一次put时会进入 initTable方法 initTable是一个死循环方法

      • 第一步: 判断sizeCtl是否等于0
        • 不等于0说明有别的线程在进行初始化,直接线程 Thread.yield() 让出cpu
      • 第二步:sizeCtl==0,使用 CAS将sizeCtl改为-1, 如果CAS成功则进入逻辑 else if (U.compareAndSwapInt(this, SIZECTL, sc, -1))
        • 如果设置失败 则进入下一次循环
      • 第三步: CAS成功后先判断 tab是否为空,长度是否为0 if ((tab = table) == null || tab.length == 0)
        • 如果判断不满足,则证明数组已经初始化,直接break退出操作
      • 第四步: 执行初始
        • 创建局部数组变量, 根据sc进行初始化数组长度,如果没指定就是16
        • 创建完成后将数组赋值给全局变量
      • 第五步: 计算新的SC
        • n - (n >>> 2) 算出来相当于 n减四分之一n = 3/4N 相当于 n*0.75
        • break

      # put方法解读

      死循环判断, 使用CAS+自旋的方式进行put

      1. 如果没有初始化table就进行初始化
      2. 初始化完毕之后再次进入循环,然后计算hash
      3. 根据算出来的hash判断位置是是否为空
        1. 为空直接CAS赋值,如果cas失败进入再次循环
        2. 如果目标位置为MOVE值,说明这个数组正在扩容 MOVED是个常量= -1
      4. 如果下标位置有元素了, 那么加sync锁主这个位置的对象(只锁对应桶位置)
      5. 如果加sync锁成功,先判断这个位置的对象是不是还是原来那个对象(有可能已经成树了)
      6. 先判断hash值是否大于0,大于0=链表结构,否则树结构
        1. 如果是链表结构就进行 追加操作或者修改操作
        2. 如果是树就进行树的操作
      7. 元素添加完成之后 判断链表长度是否 >=8

        1. =8进入转换树操作, 变树操作中,如果数组长度小于64 不会进行变树(走扩容机制)

      8. 维护集合长度
      9. 集合长度是否达到阈值,达到了就进行扩容
