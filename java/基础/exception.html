          # 2. Error 异常

          error 是系统错误,不能进行捕捉,不能通过程序来进行错误处理. 常见的比如 OutOfMemoryError 之类，都是 Error 的子类。

          # 3. Exception 异常

          又叫可捕捉异常,其中又分为 运行时异常和编译时异常
          可以 try,catch 进行捕捉,或者通过程序优化来避免.

          • 运行时异常: RuntimeException
          • 编译时异常: 在源代码编译期间必须进行处理

          # 4. try-catch-finally 的执行过程？如果存在 return 语句

          • Try:
            • try 里面可以进行 return
            • try 里面的语句出错了可以被 catch 捕获到
          • Catch:
            • try 捕获到异常后悔进入这个方法块
            • 这个里面也可以进行 return
          • Fianlly:
            • 无论如何这个里面的方法都会执行
            • 在 return 前,Catch 之后执行
            • 可以进行return .如果在 fianlly 中写了 return,这个方法返回的结果永远都是 fianlly 中返回的结果
