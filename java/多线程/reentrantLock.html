      # 1. ReentrantLock 原理概述

      三个内部类

      ReentrantLock内部有三个内部类Sync(同步器),NonfairSync（非公平锁）,FairSync（公平锁），另外实现了Lock接口。

      Sync(同步器)继承了AQS类（AbstractQueuedSynchronizer）； NonfairSync（非公平锁）继承了 Sync类； FairSync（公平锁）继承了Sync类。

      所以ReentrantLock是基于AQS实现的，AQS是java并发包的基础类。里面维护者一个同步状态state，和一个同步队列FIFO以及操作state和同步队列的方法。

      如果有一个线程过来尝试用ReentrantLock的lock()方法进行加锁，会发生什么事情呢？

      AQS对象内部有一个核心的变量叫做state，是int类型的并且加了volatile关键字，代表了加锁的状态。初始状态下，这个state的值是0。

      AQS内部还有一个关键变量ExclusiveOwnerThread，用来记录当前加锁的是哪个线程，初始化状态下，这个变量是null

      # 1.1. ReentrantLock 特性

      • 可重入锁
      • 可响应中断
      • 可尝试加锁
      • 限时等待尝试加锁
      • 公平锁、非公平锁
      • 与Condition信号量结合使用

      # 3.1. 五种状态

      负值表示结点处于有效等待状态，而正值表示结点已被取消。所以源码中很多地方用>0、<0来判断结点的状态是否正常。

      名称 对应号 说明
      EXCLUSIVE 0 默认状态
      CANCELLED 1 表示当前结点已取消调度。当timeout或被中断（响应中断的情况下）
      会触发变更为此状态，进入该状态后的结点将不会再变化。
      SIGNAL -1 表示后继结点在等待当前结点唤醒。
      后继结点入队时，会将前继结点的状态更新为SIGNAL。
      CONDITION -2 表示结点等待在Condition上，当其他线程调用了Condition的signal()方法后，CONDITION状态的结点将从等待队列转移到同步队列中，等待获取同步锁。
      PROPAGATE -3 共享模式下，前继结点不仅会唤醒其后继结点，同时也可能会唤醒后继的后继结点。

      # 3.2. AQS模式

      AQS 有两种模式

      • 一种是独占模式 EXCLUSIVE ，另外一种是
      • 共享模式 SHARED；而ReentrantLock是独占模式。

      • SHARED 共享模式，表示可以多个线程同时执行

      • EXCLUSIVE 独占模式，表示只有一个线程能执行

      • CANCELLED：1 取消状态，表示这个结点被取消了，可能是被主动取消或者超时，后续这个结点会被踢出队列

      • SIGNAL： -1 通知后继结点，表示这个结点执行完成以后，需要通知唤醒后继的结点

      • CONDITION：-2 说明这个结点因为被某一个condition挂起了

      • PROPAGATE：-3 在共享模式下，下一次获取锁后可以无限传播（不太懂这个意思，后续再好好理解）

      除了以上常量，再看下几个变量

      • waitStatus 当前结点的状态，默认是0，可以是CANCELL、SIGNAL 等
      • prev 前继结点
      • next 后继结点
      • thread 对应的线程
      • nextWaiter 下一个等待condition的结点
      • state 状态；是一个int值，表示当前线程占用资源的数量；0表示空闲，没有线程占用；ReentrantLock的state表示线程重入锁的次数

      在AQS里面的方法分成两类：

      • acquire()、acquireInterruptibly()、release() 不以share结尾的方法名；
      • acquireShared()、acquireSharedInterruptibly()、releaseShared() 以share结尾的方法名；
        顾名思义， 以share结尾的方法用于共享模式，不以share结尾的方法用于独占模式。

      # 4.3. 获取锁

      public final void acquire(int arg) {
    if (
    	//尝试获取锁(或锁重入),获取到了返回true ,否则false
    	!tryAcquire(arg) &&
        // 如果没有获取到锁,进行排队流程
        // 1.addWaiter将线程装入CHL链表(没有就创建一个头部为空对象的链表)
        // 2.acquireQueued()进行排队
        // acquireQueued 这个方法获取到锁之后会直接执行,没有获取到就会被挂起
        acquireQueued(addWaiter(Node.EXCLUSIVE), arg))
        selfInterrupt();
}

      # 4.4. tryAcquire(arg)

      进行获取锁,如果获取锁成功就返回true ,否则false

      final boolean nonfairTryAcquire(int acquires) {
    final Thread current = Thread.currentThread();
    //getState的 初始值是0
    int c = getState();
    //在state等于0的时候(代表当前没有线程在执行)
    if (c == 0) {
        //如果成功改变STATE的状态,就设置当前线程独占
        if (compareAndSetState(0, acquires)) {
            setExclusiveOwnerThread(current);
            return true;
        }
    }
    //state不等于0,代表有线程已经锁定了
    //1.判断当前线程是否已经获取到锁了(重入锁逻辑)
    else if (current == getExclusiveOwnerThread()) {
        // 重入次数+1
        int nextc = c + acquires;
        //如果重入次数小于0,就是有问题
        if (nextc < 0) 
            throw new Error("Maximum lock count exceeded");
        
        //将重入次数设置到state上
        setState(nextc);
        return true;
    }
    return false;
}

      # 4.5. acquireQueued

      final boolean acquireQueued(final Node node, int arg) {
    boolean failed = true;
    try {
        boolean interrupted = false;
        for (;;) {
            final Node p = node.predecessor();
            // 前一个等于头节点 并且 我获取到了锁
            if (p == head && tryAcquire(arg)) {
                // 将当前线程调整为头
                setHead(node);
                p.next = null; //去掉P的其他引用, help GC
                failed = false;
                return interrupted;
            }
            // 否则 检查排队的前一个节点是不是SINGLE状态,
            // 如果是就将当前线程挂起
            // 如果不是 无线循环获取前一个节点的状态,知道前置节点为SINGLE
            if (shouldParkAfterFailedAcquire(p, node) &&
                parkAndCheckInterrupt())
                interrupted = true;
        }
    } finally {
        if (failed)
            //取消获取锁
            cancelAcquire(node);
    }
}

      # 4.5.1. addWaiter(Node.EXCLUSIVE))

      将当前任务加入排队队列中,

      如果队列没有创建就创建一个再加入

      private Node addWaiter(Node mode) {
    //创建一个Node  mode=EXCLUSIVE
    Node node = new Node(Thread.currentThread(), mode);
   	// tail第一次肯定是空,
    Node pred = tail;
    if (pred != null) {
        //将对象追加到CLH链表尾部
        node.prev = pred;
        if (compareAndSetTail(pred, node)) {
            pred.next = node;
            return node;
        }
    }
    //将节点加入链表中
    enq(node);
    return node;
}

      # 4.6. enq - 誓死要加入到CLH队列中

      // 参数为当前线程创的Node
private Node enq(final Node node) {
    // 死循环
    for (;;) {
        //第一次一定等于null
        Node t = tail;
        //1.创建一个空节点,即为头又为尾
        if (t == null) {
            if (compareAndSetHead(new Node()))
                //进过这一轮操作,head和tail都是一个空的Node
                tail = head;
        } else {	//进行链路组装,无限循环,必须加入到链路中
            //2. 将node的千一个节点设置为head
            node.prev = t;
            if (compareAndSetTail(t, node)) {
                //将head的下一个节点设置为node
                t.next = node;
                return t;
            }
        }
    }
}

      # 4.7. shouldParkAfterFailedAcquire - 排队失败后挂起

      获取前一个节点的状态,

      如果前一个节点是Single状态就放true,

      否则将前一个节点改成SINGLE专业

      private static boolean shouldParkAfterFailedAcquire(Node pred, Node node) {
    int ws = pred.waitStatus;
    // 这个状态代表后继节点等待前置节点的唤醒
    if (ws == Node.SIGNAL)
        return true;

    // 大于0代表这个节点状态为非正常状态(取消状态)
    if (ws > 0) {
        do {
            //无线回溯,,直到找到一个正常状态的节点,然后在进行下一步
            node.prev = pred = pred.prev;
        } while (pred.waitStatus > 0);
        // 如果找到了就把当前节点链接到正常的节点后面
        pred.next = node;
    } else {
        //状态小于0代表在有效状态
        //将前一个节点改为 SIGNAL状态
        compareAndSetWaitStatus(pred, ws, Node.SIGNAL);
    }
    return false;
}

      # 5. 释放锁

      # 5.1. 释放流程 - release()

      • 将当前线程的重入次数-1 , 如果减成了0就将AQS中的独占线程标记设置为空
        • 如果state大于0 代表当前线程还拥有锁, 需等待所有重入锁释放完毕才会真正释放线程
      • 当重入完全释放之后开始唤醒其他线程,头节点不为空的情况下,(调用unparkSuccessor方法), 如果创建了CLH队列之后头节点不会为空
        • 判断头节点状态,状态为挂起时(小于0) ,就将状态改为初始状态
        • 获取头节点的下一个节点
          • 不等于空 : 唤醒
          • 等于空或者状态不正常: 尝试获取有没有下一个节点,如果有就一从后往前遍历,直到找到距离头节点最近的状态正常的节点,将其唤醒

      # 6. 可重入原理

      **原理：**检查state字段，若为0，表示锁未被占用，那么尝试占用，若不为0，检查当前锁是否被自己占 用，若被自己占用，则更新state字段，表示重入锁的次数。如果以上两点都没有成功，则获取锁失败，返回false。
